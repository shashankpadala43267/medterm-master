MEDTERM MASTER PWA + JSON QUESTION SYSTEM

UPLOAD ALL OF THESE ITEMS TO THE ROOT OF THE SAME GITHUB REPOSITORY:

index.html
questions.json
manifest.json
service-worker.js
question-importer.html
question-template.csv
icons/
  icon-192.png
  icon-512.png

IMPORTANT
- Keep the filenames exactly the same.
- Do not upload only index.html. The app now loads its 673 questions from questions.json.
- GitHub Pages must stay enabled on the main branch and root folder.
- The Firebase project and Firestore rules remain the same.

PWA
- Android/desktop browsers may show an install option.
- On iPhone/iPad, open the live site in Safari, tap Share, then Add to Home Screen.
- The local app shell and question bank are cached.
- Firebase sign-in and cloud syncing still require an internet connection.
- After replacing files, browsers may briefly show the old cached version. Refresh once or close and reopen the installed app.

QUESTION IMPORTER
1. Open question-importer.html from the live GitHub Pages website.
2. Download the CSV template or use your own CSV with these exact columns:
   question, choice1, choice2, choice3, choice4, answer, category, explanation
3. Upload the CSV or a compatible JSON file.
4. The tool validates the data and downloads a new questions.json.
5. Replace the old questions.json in GitHub.
6. Change CACHE_NAME in service-worker.js from medterm-master-v1 to medterm-master-v2 after updating questions, so installed copies refresh reliably.

VALID ANSWER FORMATS
- A, B, C, or D
- 1, 2, 3, or 4
- 0, 1, 2, or 3

TESTING
1. Open the public GitHub Pages link.
2. Confirm the home page shows 673 questions and 14 categories.
3. Start a quiz and confirm questions load.
4. Open question-importer.html and test the provided CSV template.
5. Test Add to Home Screen from Safari or an install option from Chrome.
6. Confirm Google Sign-In, progress syncing, and private analytics still work online.
